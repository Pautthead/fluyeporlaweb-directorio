## Qué es

Public APIs es una lista colaborativa de APIs gratuitas organizada por categoría.

El cuarto repo más destacado de todo GitHub con 441.000 estrellas.

Finanzas, música, clima, juegos, noticias, geolocalización, gobierno, salud y decenas de categorías más.
Todo con descripción, si requiere autenticación, si tiene CORS y si es HTTPS.

## Por qué importa

Cuando construyes algo y necesitas datos externos, la pregunta siempre es la misma: ¿dónde encuentro una API que haga esto?

Antes había que buscar en Google, probar servicios, registrarse, descubrir que tienen límites o que no son gratuitas de verdad.

Este repo resuelve eso. Lista curada, mantenida por la comunidad, con información real sobre cada API.

Es el punto de partida de cualquier developer que necesita datos sin pagar.

## Cómo usarlo

No se instala. Es una lista en Markdown en GitHub.

Abres el README y buscas la categoría que necesitas.
O usas la búsqueda de GitHub (Ctrl+F en el README) para encontrar lo que buscas.

También hay una API de la propia lista si quieres integrarla programáticamente:

```
https://api.publicapis.org/entries
```

## Categorías destacadas

- **Finanzas:** cotizaciones, criptomonedas, datos bancarios
- **IA y Machine Learning:** modelos, clasificadores, generadores de texto
- **Música:** letras, metadatos, streams
- **Clima:** temperatura, predicciones, datos históricos
- **Juegos:** datos de Steam, Minecraft, Pokémon
- **Gobierno:** datos públicos de España, UE y otros países
- **Redes sociales:** estadísticas, embeds, publicaciones

## Casos de uso

- Encontrar una API de clima gratuita para tu app en 2 minutos.
- Buscar datos de criptomonedas sin pagar suscripción.
- Enriquecer tu proyecto con datos de terceros sin montar infraestructura propia.
- Prototipar rápido con datos reales antes de invertir en APIs de pago.
- Descubrir APIs que no sabías que existían.

## Limitaciones honestas

Es una lista, no una plataforma. No filtra ni prueba las APIs automáticamente.

Algunas entradas pueden estar desactualizadas si el servicio original cambió sus condiciones.

La calidad y los límites de cada API dependen de quien la mantiene, no de este repo.

## Recursos

- Repositorio en GitHub (link arriba)
